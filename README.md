# Qmooni (퀴문이)

RAG 기반 문서 챗봇 & 자동 퀴즈 생성 플랫폼

'퀴즈 + 문서', '퀴즈 + 문의(질문)'의 중의적 의미를 담은 이름으로, 사용자가 업로드한 문서를 기반으로 챗봇 기능을 지원하고 퀴즈를 자동으로 생성하여 효율적인 학습과 복습을 돕는 웹 기반 지능형 학습 플랫폼입니다.

## 프로젝트 개요

- **기간**: 2025.03 ~ 2025.06
- **인원**: 2명
- **담당**: UI/UX 설계 · Frontend · 인증 시스템 · 주제 생성 및 문서 업로드 기능


## 핵심 기능

### 문서 기반 챗봇 (LangGraph + RAG)
- 업로드한 문서(PDF, PPTX, DOCX) 기반 질의응답
- 요약본 / 원문 검색 판단 후 RAG 기반 응답 생성
- 사용자가 요약된 정보를 원할 경우 재사용하여 OpenAI API 호출 최소화

### 자동 퀴즈 생성
- 객관식(MCQ), OX 문제 자동 생성
- 풀이 기록, 정답률, 오답 복습 기능

## 담당 역할 (Frontend 중심)

- 전체 디자인 및 UI/UX 구성
- 프론트엔드 개발 (React + Tailwind CSS)
- Google OAuth2 + JWT 사용자 인증 구현
- 주제(Topic) 생성 및 문서 업로드 기능 구현
- 백엔드 API 연동 및 명세 설계 참여

## 트러블슈팅

### 퀴즈 생성 품질 개선 (백엔드 담당)

**초기 시도**: 문서 전체 내용을 요약해 GPT-3.5-turbo + custom prompt로 MCQ/OX 문제를 자동 생성했습니다.

**한계점**: 문서 전체를 한 번에 요약하다 보니 글자 수 제한으로 문제 다양성이 부족했고, 퀴즈를 요청할 때마다 요약을 새로 생성해 API 비용이 늘고 응답 속도가 느려지는 문제가 있었습니다.

**해결책**: 문서 업로드 시점에 페이지 단위로 요약을 미리 생성하고, 원문과 요약본을 모두 FAISS에 저장해두었습니다. 이후 퀴즈 생성 시에는 저장된 요약본만 사용하도록 구조를 변경해 응답 속도와 비용 문제를 함께 개선했습니다.
